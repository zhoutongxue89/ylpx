Kubernetes 学习之路

从入门到实战的全方位指南

📋 什么是 Kubernetes？

Kubernetes（简称 K8s）是一个开源的容器编排平台，用于自动化容器化应用的部署、扩展和管理。它由 Google 开发，现已成为云原生时代的核心技术之一。

为什么学习 Kubernetes？
在微服务架构和云原生时代，Kubernetes 已成为容器编排的事实标准。掌握 K8s 能让你更好地理解现代应用的部署和管理，提升职场竞争力。

📖 第一阶段：Docker 基础

1Docker 核心概念

在学习 K8s 之前，需要先掌握容器技术。Docker 是最流行的容器运行时。

需要掌握的内容：

  • 镜像（Image）：容器模板，类似于面向对象中的类
  • 容器（Container）：镜像的运行实例，类似于对象
  • 仓库（Registry）：存储和分发镜像的地方（如 Docker Hub）
  • Dockerfile：定义镜像构建脚本

常用命令：

# 镜像操作 docker pull nginx:latest # 拉取镜像 docker images # 查看镜像列表 docker rmi nginx # 删除镜像 # 容器操作 docker run -d -p 80:80 nginx # 运行容器 docker ps # 查看运行中的容器 docker ps -a # 查看所有容器 docker stop container_id # 停止容器 docker rm container_id # 删除容器 # 构建镜像 docker build -t myapp:v1 . # 构建镜像 docker tag myapp:v1 registry/myapp:v1 # 打标签 docker push registry/myapp:v1 # 推送镜像

📖 第二阶段：Kubernetes 核心概念

1Kubernetes 架构

K8s 采用主从架构，主要包含以下组件：

  • Master 节点：控制平面，负责集群管理
    • API Server：提供 REST API
    • Scheduler：负责 Pod 调度
    • Controller Manager：控制器管理
    • etcd：分布式存储
  • Worker 节点：工作节点，运行容器
    • Kubelet：与 Master 通信
    • Kube Proxy：网络代理
    • Container Runtime：容器运行时（Docker/Containerd）

2核心资源对象

Pod

Pod 是 K8s 最小的调度单元，一个 Pod 可以包含一个或多个容器。

apiVersion: v1 kind: Pod metadata: name: my-pod labels: app: myapp spec: containers: - name: nginx image: nginx:latest ports: - containerPort: 80

Deployment

Deployment 管理 Pod 的部署，支持滚动更新和回滚。

apiVersion: apps/v1 kind: Deployment metadata: name: my-deployment spec: replicas: 3 selector: matchLabels: app: myapp template: metadata: labels: app: myapp spec: containers: - name: nginx image: nginx:latest ports: - containerPort: 80

Service

Service 为 Pod 提供稳定的访问入口，屏蔽 Pod 的动态性。

apiVersion: v1 kind: Service metadata: name: my-service spec: selector: app: myapp ports: - port: 80 targetPort: 80 type: ClusterIP # 或 NodePort, LoadBalancer

ConfigMap & Secret

ConfigMap 用于存储配置，Secret 用于存储敏感信息。

Ingress

Ingress 提供 HTTP/HTTPS 路由功能。

3常用命令

# 资源操作 kubectl get pods # 查看 Pod kubectl get pods -o wide # 详细信息 kubectl describe pod my-pod # 查看 Pod 详情 kubectl logs my-pod # 查看日志 kubectl exec -it my-pod -- /bin/sh # 进入容器 # 部署管理 kubectl apply -f deployment.yaml # 创建/更新资源 kubectl delete -f deployment.yaml # 删除资源 kubectl rollout status deployment/my-deployment # 查看滚动更新状态 kubectl rollout undo deployment/my-deployment # 回滚 # 扩缩容 kubectl scale deployment my-deployment --replicas=5 # 调试 kubectl port-forward my-pod 8080:80 # 端口转发 kubectl get events # 查看事件 kubectl top nodes # 查看资源使用

📖 第四阶段：项目实战

1本地搭建集群

  • Minikube：单机版 K8s，适合学习和测试
  • Kind：使用 Docker 容器运行 K8s 节点
  • K3s：轻量级 K8s，适合边缘计算和资源受限环境
  • k0s：另一个轻量级发行版

2云原生部署实践

推荐在云平台上进行实战：

  • 阿里云 ACK：阿里云容器服务
  • 腾讯云 TKE：腾讯云容器服务
  • AWS EKS：亚马逊云容器服务
  • GKE：Google Kubernetes Engine
  • Azure AKS：Azure Kubernetes Service

3实战项目推荐

  1. 将 Spring Boot 应用部署到 K8s

  2. 搭建 GitLab + Jenkins/ArgoCD CI/CD 流水线
  3. 部署微服务架构（Spring Cloud + K8s）
  4. 搭建日志/监控系统（Prometheus + Grafana + EFK）
  5. 配置高可用集群

💡 学习建议

1. 多动手实践
纸上谈兵不可取，一定要自己搭建集群，部署应用，遇到问题自行排查。
2. 理解原理
不仅要知道怎么用，还要理解为什么这样设计。了解 K8s 的架构和设计理念。
3. 参与社区
加入 K8s 社区，参与开源项目，关注最新动态。
4. 考取认证
CKA（Kubernetes 管理员认证）和 CKAD（Kubernetes 应用开发者认证）值得考虑。
5. 持续学习
K8s 版本更新频繁，要保持学习，跟上技术发展。

🎯 认证路径

CKA - Certified Kubernetes Administrator

管理员认证，考察集群管理能力

考试时长：2小时 题目数量：15-20题 通过分数：66%

CKAD - Certified Kubernetes Application Developer

开发者认证，考察应用开发部署能力

考试时长：2小时 题目数量：15-20题 通过分数：66%

CKS - Certified Kubernetes Security Specialist

安全专家认证（需要先通过 CKA）

考试时长：2小时 题目数量：15-20题 通过分数：67%

📈 技术生态

Kubernetes 已经形成了一个庞大的技术生态，以下是核心组件：

  • 容器运行时：Docker、Containerd、Cri-O
  • 网络插件：Calico、Flannel、Cilium、Weave Net
  • 存储插件：Ceph、GlusterFS、Longhorn
  • 服务网格：Istio、Linkerd、Envoy
  • CI/CD：Jenkins、GitLab CI、ArgoCD、Flux
  • 监控：Prometheus、Grafana、ELK/EFK
  • 日志：Fluentd、 Loki、Jaeger
  • 包管理：Helm、ArgoCD Applications

🚀 总结

Kubernetes 是现代云原生技术的基础设施，学习曲线相对陡峭，但只要循序渐进，多动手实践一定能掌握。

学习顺序建议：

  1. 先学会 Docker 基础
  2. 搭建本地 K8s 集群（Minikube/Kind）
  3. 学习核心资源对象（Pod、Deployment、Service）
  4. 学习进阶内容（存储、网络、安全）
  5. 实战项目锻炼
  6. 考虑考取 CKA/CKAD 认证
